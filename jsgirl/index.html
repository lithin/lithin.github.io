HTML

HTML is the template of the page - it tells the browser which elements go where and what the content of the page actually is.

Inspect element allows you to easily see html of any web page. It opens if you right click on any element on the page, and select “Inspect”.

Tags are these slightly weird letters or words in angle brackets. Tags are what HTML consists of, and they usually go in pairs: with opening tag and closing tag.

CSS

CSS is a description of how things should look in browser - colour of text, font size, background colour, … It’s literally the “Style Sheet”

Inspect element allows us to see styles as well as html. Every element will have a list of styles that are applied. The styles that aren't valid or are over-ridden by another declaration are crossed over.

You may notice user agent stylesheets - these are default styles of elements in browsers (e.g. a paragraph normally has a bit of a margin at the bottom).

Here are a few very commonly used style declarations:

  • font-size: size of the text
  • font-family: type of text such as Times New Roman or Helvetica
  • font-weight: bold, thin etc
  • margin: “outter” spacing around element
  • padding: “inner” spacing around element
  • color: colour of the text
  • background: background of the element - can be a colour or an image
  • border: border of the element; it sits between margin and padding

Every html element can have a class that can be then used to style this element or to get it from javascript application.

JS

You can interact with javascript in browser through the Console. Press alt + cmd + j to open it - or inspect any element and choose the Console tab in dev tools.

Libraries and frameworks

When coding in javascript, you'll hear words like libraries and frameworks very often.

Library is a piece of code that helps us with our own programming. It’s usually relatively simple a does one thing.

Framework is also a piece of code but is really complex. We use it to build applications with because it does a lot of work for us.

How the internet works

Browser or a mobile application is usually called a "client". Opposed to it is the server which a client connects to. Server lives "on the internet" and we can access it from anywhere when we're online.

Client code is the code that runs in the browser - it displays the website and gives user the ability to interact with content.

Server is the computer that clients connect to to see websites and get information.

Server-side application manages data, saves them and updates them in the database, or gets them from the database for us. It can send emails, collect and process data.

Database is a data storage that keeps any information we save into it for as long as we want to keep it there.
